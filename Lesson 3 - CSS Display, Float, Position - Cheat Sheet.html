CSS Cheat Sheet, Display / Float / Position

1. Display

Property / Value Meaning Behavior Example
display: block Element behaves as block Starts on new line, full width, can set width/height <div style="display:block">...</div>
display: inline Inline element No new line, width equals content, cannot set width/height <a style="display:inline">Link</a>
display: inline-block Hybrid of inline + block No new line, but width/height allowed <div style="display:inline-block; width:100px;"></div>
Default block elements Structural tags Take full width div, p, ul, li, h1-h6
Default inline elements Text-level elements Flow inside text span, a, b, i, u, img

2. Float

Float Feature Description Effect Example
float: left Push element to the left Text wraps around the floated element .img { float:left }
float: right Push element to right side Used for sidebar layouts .sidebar { float:right }
Both image + text floated If both float, text no longer wraps normally Each floated element takes its own side img {float:left}, p {float:right}
Two-column layout Using float for layout Sidebar left, main content right .sidebar {float:left}, .main {float:right}
clear: both Stops floating and forces next line Prevents overlap of footer/content .footer { clear:both }
No float Natural document flow Elements stack normally display:block
Inline-block instead of float Alternative layout technique Elements appear side by side without wrapping text .box {display:inline-block}

3. Position

position Value Description Behavior Example
static Default position Normal flow, offsets ignored .box { position:static }
relative Moves relative to its normal place Space remains reserved .box { position:relative; top:20px; }
absolute Removed from flow Positioned relative to nearest positioned parent .child { position:absolute; top:30px; }
fixed Attached to viewport Stays visible when scrolling .menu { position:fixed; bottom:10px; }
sticky Behaves as relative until threshold Then sticks to top like fixed .header { position:sticky; top:0; }
Absolute with container First positioned ancestor becomes reference Parent must use relative/absolute/fixed/sticky .parent {position:relative}, .child {position:absolute}
