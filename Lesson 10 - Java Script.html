What is JavaScript

JavaScript is a programming language that makes web pages interactive. It can respond to clicks, show messages, validate forms, create animations, games, and more.

Script Placement Guide

JavaScript code is written inside a special tag:

<script> // JavaScript code goes here </script>

We can place scripts:

  • inside the head
  • inside the body
  • in several places in the page if needed

Most developers place scripts at the end of the body so the page is fully loaded before the code runs.

<body> <h1 id="title">Hello</h1> <script> const h = document.getElementById("title"); h.innerText = "Loaded"; </script> </body>

External JavaScript file

If the same JavaScript code is needed in several pages, it is better to place it in an external file with a js extension.

<script src="file.js"></script>

This script file can then be reused in many HTML pages.

Running JavaScript with a Button

JavaScript does not have to run when the page loads. We can run code when the user clicks a button using the onclick attribute.

<body> <h1>This is a JavaScript example</h1> <p>This is a JavaScript content</p> <button onclick="alert('Hello class')"> Click Me </button> </body>

When the user clicks the button, the alert appears. This is our first example of making the page interactive.

Using getElementById

In this example we first find the button in the page using getElementById, then we attach a click event to it.

<h1>This is a JavaScript example</h1> <p>This is a JavaScript content</p> <button id="hello">Click Me</button> <script> document.getElementById("hello").onclick = function () { alert("Hello Class"); }; </script>

getElementById finds the element, and onclick defines what happens when the button is clicked.

Writing to an HTML element

In this example we change the content of a paragraph when the user clicks the button.

<h1>This is a JavaScript Example</h1> <p id="myText"></p> <button id="hello">Click Me</button> <script> document.getElementById("hello").onclick = function () { document.getElementById("myText").innerHTML = "Hello"; }; </script>

getElementById finds the paragraph, and innerHTML sets its content.

Writing HTML using JavaScript

Here we use JavaScript to insert real HTML into the page, not just plain text.

<h1>This is a JavaScript Example</h1> <p id="myText"></p> <button id="hello">Click Me</button> <script> document.getElementById("hello").onclick = function () { document.getElementById("myText").innerHTML = "<h3>Hello</h3>"; }; </script>

Using innerHTML allows us to create new elements dynamically.

Variables

A variable is a container that stores a value. JavaScript variables do not have a fixed type. The type depends on the value stored in them.

There are four ways to declare a variable:

Keyword Can reassign value? Can redeclare? Scope Notes
var Yes Yes Function scope Old syntax, can cause bugs
let Yes No Block scope Recommended for variables that change
const No No Block scope Recommended for values that should not change
Implicit (no keyword) Yes Yes Global Not recommended

Most modern code uses let and const, and avoids var.

Redeclaring variables: let vs var

In JavaScript, variables behave differently when we try to declare them again.

Using let

let x; // undefined let x = "John Doe"; // SyntaxError, x is already declared

Variables declared with let cannot be redeclared in the same scope.

Using var

var x; // undefined var x = "John Doe"; var x = 0; // allowed var x; // still allowed, value remains 0

Variables declared with var can be redeclared. This can cause bugs, which is why modern JavaScript prefers let and const.

Block scope: let vs var

A block is anything inside curly braces { }. Variables declared with let exist only inside that block. Variables declared with var ignore the block.

{ let x = 2; } // x can NOT be used here
{ var x = 2; } // x CAN be used here

This shows the main difference: let is block-scoped, while var is not.

Hoisting: var vs let

Hoisting means that variable declarations are moved to the top of their scope before the code is executed.

Hoisting with var

Variables declared with var are hoisted and initialized with undefined. This means you can use the variable before it is declared.

carName = "Volvo"; var carName;

The code above works, although it is not recommended.

Hoisting with let

Variables declared with let are also hoisted, but they stay in a Temporal Dead Zone until the declaration line is reached.

Using the variable before it is declared will cause a ReferenceError:

carName = "Volvo"; let carName; // Uncaught ReferenceError ReferenceError: Cannot access 'carName' before initialization

So while both var and let are hoisted, only var can be accessed before its declaration.

Variables and data types

Variables in JavaScript can store different types of data such as: numbers, strings, booleans, arrays, and objects.

// Number var length = 16; // String var lastName = "Johnson"; // Array var cars = ["Saab", "Volvo", "BMW"]; // Object var person = { firstName: "John", lastName: "Doe" };

Array Example in the page:

<!DOCTYPE html> <html> <body> <p id="demo"></p> <script> var cars = ["Saab", "Volvo", "BMW"]; document.getElementById("demo").innerHTML = cars[0]; </script> </body> </html>

In this example we use an array, and display the first value in it.

Object Example in the page:

<!DOCTYPE html> <html> <body> <p id="demo1"></p> <p id="demo2"></p> <script> var person = { firstName: "John", lastName: "Doe" }; // Dot notation document.getElementById("demo1").innerHTML = "First name: " + person.firstName; // Bracket notation document.getElementById("demo2").innerHTML = "Last name: " + person["lastName"]; </script> </body> </html>

Dot notation is most common. Bracket notation is useful when the property name comes from a variable.

Comments

Comments are text inside the code that JavaScript ignores. We use them to explain what the code does.

There are two kinds of comments:

Single-line comment

// this is a one line comment

Block comment

/* this is a block of comments */

Comments are useful for documentation, reminders, and hiding code temporarily.

Conditional statements

Conditional statements let us run code only when a condition is true.

let grade = 85; if (grade >= 90) { console.log("Excellent"); } else if (grade >= 70) { console.log("Good"); } else { console.log("Needs improvement"); }

In this example, the message depends on the value of grade.

Loops

Loops let us repeat code multiple times.

for loop

for (let i = 0; i < 5; i++) { console.log("Number:", i); }

Runs 5 times, from 0 to 4.

while loop

let x = 0; while (x < 3) { console.log("x is", x); x++; }

Repeats while the condition is true.

do-while loop

let y = 5; do { console.log("y is", y); y++; } while (y < 5);

The code runs at least once, even if the condition is false.

User Interaction with prompt()

We can get input from the user using the prompt dialog. The prompt returns a string, or null when the user presses Cancel. 

let name = window.prompt("Please enter your name", "Guest");

if (name === null) {
    document.write("User pressed Cancel");
} else {
    document.write("Hello " + name);
}

Example: Read two numbers and calculate

In the next example we:

  • Ask for two integers
  • Stop the program if Cancel is pressed
  • Convert input strings to numbers
  • Avoid division by zero
let firstNumber = window.prompt("Enter integer", "0"); if (firstNumber === null) { document.write("Cancel button was pressed"); } else { let secondNumber = window.prompt("Enter integer", "0"); if (secondNumber === null) { document.write("Cancel button was pressed"); } else { let n1 = parseInt(firstNumber); let n2 = parseInt(secondNumber); if (n2 === 0) { document.write("Can not divide by zero"); } else { let multiply = n1 * n2; let division = n1 / n2; document.write(n1 + " * " + n2 + " = " + multiply + ""); document.write(n1 + " / " + n2 + " = " + division + ""); } } }

Important notes:

  • prompt always returns text
  • We use parseInt to convert to an integer
  • Cancel returns null
  • We must check for division by zero

Using CSS with JavaScript

JavaScript can also change the style of elements on the page. The best way to do this is by adding or removing CSS classes.

Example:

<!-- HTML -->
<div id="messageBox">Click a button</div>

<!-- CSS -->
<style>
    .success {
        background-color: lightgreen;
        color: darkgreen;
        padding: 10px;
        border: 2px solid darkgreen;
    }

    .error {
        background-color: lightpink;
        color: darkred;
        padding: 10px;
        border: 2px solid darkred;
    }
</style>

<!-- JavaScript (replace all classes) -->
document.getElementById("messageBox").className = "success";

In this example:

  • The CSS classes describe different visual states.
  • JavaScript finds the element and sets its class to success.
  • Any previous classes on the element are removed.

Adding a class instead of replacing it

Sometimes the element already has classes, and we only want to add a new one. In that case we use classList.add(): 

<!-- JavaScript (add without deleting) -->
document.getElementById("messageBox").classList.add("success");

Now the element keeps its old classes, and success is added on top of them.

  • className = "success" replaces everything.
  • classList.add("success") keeps the old classes and adds a new one.

This is the most common pattern when working with JavaScript and CSS together.

Removing a class

Sometimes we want to undo a style, for example remove success and show error instead. We use classList.remove(): 

<!-- JavaScript (remove a class) -->
document.getElementById("messageBox").classList.remove("success");

If the class exists, it is removed. If it does not exist, nothing bad happens, the code just continues.

  • classList.add("success") → adds the class
  • classList.remove("success") → removes the class

Putting it together

let box = document.getElementById("messageBox");

// show success
box.classList.add("success");

// later, switch to error
box.classList.remove("success");
box.classList.add("error");

This pattern is very common when showing messages, alerts, and form validation.

Selecting elements in JavaScript

JavaScript needs to select an element before it can change it. There are several ways to do this, not only by id.

1. Select by id

document.getElementById("title");

Best when the element is unique on the page.

2. Select by class name

document.getElementsByClassName("item");

Returns a list, because many elements can share the same class.

3. Select by tag name

document.getElementsByTagName("p");

Selects all elements with the same tag, for example all <p> tags.

4. Using CSS selectors (modern way)

Works the same as CSS selectors. 

document.querySelector("#menu");        // first match
document.querySelectorAll(".item");     // all matches

5. Select by name (often used in forms)

document.getElementsByName("username");

This is useful when working with form fields.

Summary:

  • getElementById - simple and very fast
  • getElementsByClassName / TagName - returns lists
  • querySelector / querySelectorAll - flexible, works like CSS
  • getElementsByName - mostly for forms

Functions and Variable Scope

A function is a block of code that runs only when it is called. It can receive values (parameters) and return a value.

Example function

function add(a1, a2) { a3 = 10; // global variable let sum = a1 + a2; // local variable (visible only inside function) return sum * factor; } var factor = 5; // global variable var res = add(3, 4) + a3; document.write(res);

Local variables

  • Defined inside a function using let, const, or var
  • Can be used only inside that function
function test() { let x = 5; }

Global variables

  • Defined outside any function, or created without a declaration keyword
  • Can be used anywhere in the script
var y = 10; function show() { document.write(y); }

⚠️ Creating variables without let/var/const automatically makes them global. This is considered bad practice.

Summary

  • Local variable: exists only inside a function.
  • Global variable: exists everywhere in the program.
  • A variable that receives a value inside a function but is not declared becomes global automatically.

Calling a function from a button

A button can call a JavaScript function using the onclick attribute. In this example the function adds two numbers.

<!-- HTML --> <input id="num1" type="text" value="5"> <input id="num2" type="text" value="3"> <button onclick="addNumbers()">Add</button> <p id="result"></p> <!-- JavaScript --> function addNumbers() { // read values from the inputs let a = parseInt(document.getElementById("num1").value); let b = parseInt(document.getElementById("num2").value); // calculate let sum = a + b; // show result in the page document.getElementById("result").innerHTML = "Result: " + sum; }

The button calls the function, the function does the calculation, and then it writes the result into the page.

Arrays

An array is an object that stores a list of values. Each value is called an element.

  • All elements have the same array name
  • Each element has an index number
  • Indexes start from 0
  • The array size grows dynamically
  • When an array is passed to a function, it can be changed inside the function

Creating arrays

You can create arrays in several ways:

let arr = new Array(3);
let arr = new Array("morning", "day", "night");
let arr = ["morning", "day", "night"];

The last version using square brackets is the most common.

Array length and indexing

let arr = ["morning", "day", "night"]; arr[3] = "other"; // array grows automatically let len = arr.length; // length of the array

Arrays in JavaScript grow automatically when you add new values.

Array methods

arr.indexOf("b"); 		Returns 1

Filling an array using a loop

let randomNumbers = new Array(); for (let i = 0; i < 10; i++) { randomNumbers[i] = i; }

This creates an array with numbers from 0 to 9.

Reading user input into an array and do calculation

function fetchInput(arr) { for (let i = 0; i < arr.length; i++) { arr[i] = parseInt(window.prompt("Enter grade:", "0")); } } function calculateAverage(arr) { let sum = 0; for (let i = 0; i < arr.length; i++) { sum = sum + arr[i]; } return sum / arr.length; } let size = parseInt(window.prompt("Enter number of grades:", "1")); let arr = new Array(size); fetchInput(arr); let avg = calculateAverage(arr);

The program asks how many grades there are, creates an array, fills it, and calculates the average.

DOM, Document Object Model

When a web page loads, the browser creates a DOM, Document Object Model. The DOM is a tree of objects that represents all the HTML elements in the page.

Each HTML element becomes a node in the DOM tree, such as:

  • document
  • html
  • head
  • body
  • elements like p, h1, div, a
  • attributes and text

JavaScript uses the DOM to:

  • Access HTML elements
  • Change their text
  • Change their HTML
  • Change styles
  • Add or remove elements

The document object

The document object represents the entire web page. It allows JavaScript to find and change elements in the HTML.

document.getElementById("content");

This code finds the element whose id is content.

HTML:

<section id="content">Hello</section>

Reading text from an element

document.getElementById("content").textContent;

This reads the plain text inside the element.

Changing text inside an element

document.getElementById("content").innerHTML = "Hello";

This replaces the content of the element with new text or HTML.

Example

<p id="content">Old text</p> <script> document.getElementById("content").innerHTML = "New text"; </script>

The paragraph will now show New text.

Selecting elements in the DOM

There are several ways to find elements in the DOM.

By ID

document.getElementById("title");

Finds one unique element.

By tag name

document.getElementsByTagName("p");

Finds all <p> elements and returns a list.

By class name

document.getElementsByClassName("intro");

Finds all elements with class intro.

Using CSS selectors

document.querySelectorAll("p.intro");

Finds all <p> elements with class intro. This works the same as CSS selectors.

Example with multiple elements

<p class="intro">First</p> <p class="intro">Second</p> <script> let items = document.getElementsByClassName("intro"); items[0].innerHTML = "Changed"; </script>

This changes the first paragraph with class intro.

CSS and DOM

JavaScript can access many elements in the page at once and change their CSS styles. This is done through the DOM.

In this example, we want to color all <p> (paragraph) elements in red.

HTML

<h1>DOM and CSS example</h1>
<p>Use CSS and DOM</p>
<p>We would like to paint all paragraph items</p>

JavaScript

let x = document.getElementsByTagName("p");
let len = x.length;

for (let i = 0; i < x.length; i++) {
    x[i].style.color = "red";
}

What happens here?

  • getElementsByTagName("p") finds all paragraph elements.
  • The result is a list (collection) of elements, an HTMLCollection object.
  • The loop goes through all paragraphs.
  • Each paragraph gets its color changed to red.

This is how JavaScript controls CSS through the DOM.
