What is a Form

A form is an HTML structure used to collect data from the user.

An interactive form allows users to enter information, choose options, and send data to another system.

A form is built from different input controls, such as text fields, checkboxes, radio buttons, and buttons.

Each control has a name and a value, which are sent when the form is submitted.

<form> <input type="text"> <input type="password"> <button>Submit</button> </form>

The <form> tag acts as a container that holds all form controls together.

1. The Input Element

<input>

The input tag collects data from the user. It has no closing tag. The behavior depends on the type attribute.

Examples:

  • text
  • password
  • checkbox
  • radio
  • number
  • email
  • submit

2. Text Inputs

<form> <p><label>First Name: <input type="text"> </label></p> </form>

type="text" creates a basic input field for entering free text such as names or titles.

3. Input with Value

value sets the initial text that appears inside the input field when the page loads.

<form> <p><label>First Name: <input type="text" value="Yogev"> </label></p> </form>

4. Input Attributes

Attributes like placeholder, maxlength, and name control the input hint, limit the text length, and define the data key.

<form> <p><label>Username: <input type="text" placeholder="Enter username" maxlength="12" name="user"> </label></p> </form>

5. Checkboxes

<form> <p><label> <input type="checkbox" value="football"> Football </label></p> <p><label> <input type="checkbox" value="basketball"> Basketball </label></p> <p><label> <input type="checkbox" value="tennis"> Tennis </label></p> </form>

Checkboxes allow selecting zero, one, or multiple options at the same time.

6. Radio Buttons

<form> <p><label> <input type="radio" name="gender"> Male </label></p> <p><label> <input type="radio" name="gender"> Female </label></p> <p><label> <input type="radio" name="gender"> Other </label></p> </form>

Radio buttons allow selecting exactly one option from a group.

All radio buttons in the group must share the same name, otherwise multiple options could be selected.

6. Form GET Method

<form action="http://example.com" method="get" target="_blank"> <label>Search: <input type="text" name="q"> </label> <input type="submit" value="Search"> </form>

When the form is submitted, the browser sends the input value as a URL parameter using the GET method.

The parameter name q is added to the URL of http://example.com.

The request is executed when the user clicks the submit button or presses Enter inside the input field.

Example URL after submitting the form with the value html:

http://example.com/?q=html

7. Form POST Method

<form action="http://example.com/submit" method="post"> <label>Username: <input type="text" name="user"> </label> <input type="submit" value="Send"> </form>

The POST method sends form data inside the request body, not in the URL.

The request is executed when the user clicks the submit button or presses Enter.

Request URL:

http://example.com/submit

Request body sent to the server:

user=Yogev

8. Novalidate

<!-- Form WITHOUT novalidate --> <form> <p>Email: <input type="email" name="email"> </p> <input type="submit" value="Submit"> </form>

By default, the browser automatically validates input fields such as email before submitting the form.

Common validation cases include required fields, email format, URL format, number ranges, date and time values, and custom patterns.

<!-- Form WITH novalidate --> <form action="http://example.com/submit" method="post" target="_blank" novalidate> <p>Email: <input type="email" name="email"> </p> <input type="submit" value="Submit"> </form>

When novalidate is used on the form, the browser disables its built in validation checks.

This allows the form to be submitted even if the email value is invalid or empty.

9. Autocomplete

<form autocomplete="on"> First name: <input type="text" name="fname"><br> Last name: <input type="text" name="lname" autocomplete="off" ><br> <input type="submit"> </form>

The autocomplete attribute controls whether the browser remembers and suggests previously entered values.

When autocomplete="on" is set on the form, the browser may suggest saved values for its inputs.

Setting autocomplete="off" on a specific input overrides the form setting and disables suggestions for that field.

10. Disabled

<input type="text" disabled>

A disabled input cannot receive focus, is skipped when using the Tab key, and its value is not sent when the form is submitted.

11. Readonly

<input type="text" readonly>

A readonly input can receive focus and is submitted with the form, but the user cannot change its value.

12. Checkbox Checked

<form> <p><label> <input type="checkbox" checked> Newsletter </label></p> <p><label> <input type="checkbox"> Promotions </label></p> </form>

The checked attribute marks a checkbox as selected by default when the page loads.

13. Radio Checked

<form> <p><label> <input type="radio" name="gender" checked> Male </label></p> <p><label> <input type="radio" name="gender"> Female </label></p> </form>

The checked attribute pre selects one radio option when the page loads.

Only one radio button in the group can be checked, because all options share the same name.

14. Select

<select> <option>Haifa</option> <option selected>Tel Aviv</option> <option>Jerusalem</option> </select>

The select element creates a dropdown list.

The selected attribute defines which option is chosen by default when the page loads.

15. Button

<form> <p><input type="button" value="Save"></p> <p><button>Save</button></p> </form>

There are two ways to create a button in HTML.

  • <input type="button"> creates a simple button using an input element.
  • <button> creates a button element that can contain text, HTML, or icons.

Both buttons look similar in the browser, but they are different HTML elements.

Important: These buttons do not submit the form by default. To submit a form, the button must be of type submit.

15. Password, Email, URL

<form> <input type="password"> <input type="email"> <input type="url"> <input type="submit"> </form>

type="password" is used to enter sensitive data. The characters are hidden and displayed as dots or stars.

type="email" is used to enter an email address. When the form is submitted, the browser checks that the value is in a valid email format.

type="url" is used to enter a web address. The browser validates that the value looks like a valid URL, similar to email validation.

16. Tel

<input type="tel" placeholder="050-1234567">

type="tel" is used for phone numbers and may trigger a numeric keypad on mobile devices.

17. Number

<input type="number" min="0" max="120">

type="number" allows entering numeric values and supports limits such as min and max.

18. Range

<input type="range" min="0" max="100" value="50">

type="range" displays a slider for selecting a numeric value within a range.

19. Color

<input type="color">

type="color" opens a color picker and returns the selected color value.

24. File Upload

<form> <p><label>Upload file: <input type="file" name="myfile"> </label></p> <p><label>Upload multiple files: <input type="file" name="myfiles" multiple> </label></p> <input type="submit" value="Send"> </form>

The file input allows the user to select a file from their computer.

Using the multiple attribute allows selecting more than one file.

25. Datalist

<input list="browsers"> <datalist id="browsers"> <option value="Internet Explorer"> <option value="Firefox"> <option value="Chrome"> <option value="Opera"> <option value="Safari"> </datalist>

A datalist provides a predefined list of suggested values for an input field, while still allowing the user to type any value.

The list attribute of the input must match the id of the datalist for the suggestions to appear.

26. Required

<input type="email" required>

The required attribute forces the user to fill in the field before the form can be submitted.

27. Autofocus

<input type="text" autofocus>

The autofocus attribute automatically places the cursor inside the input field when the page loads.

28. Pattern

<input type="text" pattern="\d{9}" required>

The pattern attribute defines a rule using a regular expression that the input value must match.

In this example, the input must contain exactly 9 digits, otherwise the browser will block form submission.
