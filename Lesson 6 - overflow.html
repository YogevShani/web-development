CSS Overflow Comparison

General Explanation

Each container below is only 300px wide, but the three boxes inside total more than 600px in width. This allows us to clearly see how each overflow value behaves.

How the Overflow Example Is Built

Before we look at the different overflow behaviors, we first need to understand the basic structure that is used in all examples.

HTML Structure


<div class="container"> 
  <div class="box">Box 1</div>
  <div class="box">Box 2</div>
  <div class="box">Box 3</div>
</div>

The container holds three boxes. The boxes are placed next to each other in one row.

Basic CSS Used


.container {
  width: 300px;
}

.box {
  display: inline-block;
  width: 200px;
}

The container width is only 300px, but each box is 200px wide. Three boxes together are wider than the container, so overflow is guaranteed to happen. This allows us to clearly demonstrate how each overflow mode behaves.

Explanation:
Scroll bars appear only when needed. If the content fits, there will be no scroll bar. If it overflows, scrolling becomes available automatically.

Summary

  • visible lets content escape outside the container
  • hidden cuts off overflowing content completely
  • scroll always shows scroll bars
  • auto shows scroll bars only when needed
