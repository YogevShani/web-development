Bootstrap, Step 1, Getting Started

1. What Is Bootstrap

Bootstrap (formerly Twitter Bootstrap) is a free and open-source CSS framework directed at responsive, mobile-first front-end web development.
It contains HTML, CSS and (optionally) JavaScript-based design templates for typography, forms, buttons, navigation, and other interface components.

As of May 2023, Bootstrap is the 17th most starred project (4th most starred library) on GitHub, with over 164,000 stars. According to W3Techs, Bootstrap is used by 19.2% of all websites.

Instead of writing everything from scratch, we use built-in Bootstrap classes.

2. Adding Bootstrap to an HTML Page

The easiest way to add Bootstrap is by using a CDN (Content Delivery Network) link inside the <head> tag.
1. The file is stored on powerful servers around the world
2. The website downloads the file directly from the internet
3. As a developer, you do not need to download or install anything manually 

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">

Once this line is added, Bootstrap classes are ready to use.

3. Your First Bootstrap Example

Below is a simple example of a Bootstrap styled button. No custom CSS is needed. 

<button class="btn btn-primary">Click Me</button>

4. Why This Works, The Real CSS Behind btn-primary

When we write this: 


<button class="btn btn-primary">Click Me</button>

The styling does NOT come from our file. It comes from the Bootstrap CSS file that we loaded using the CDN.

Here is the REAL CSS that Bootstrap uses for btn-primary (original version from the Bootstrap file): 


.btn-primary{
  --bs-btn-color:#fff;
  --bs-btn-bg:#0d6efd;
  --bs-btn-border-color:#0d6efd;

  --bs-btn-hover-color:#fff;
  --bs-btn-hover-bg:#0b5ed7;
  --bs-btn-hover-border-color:#0a58ca;

  --bs-btn-focus-shadow-rgb:49,132,253;

  --bs-btn-active-color:#fff;
  --bs-btn-active-bg:#0a58ca;
  --bs-btn-active-border-color:#0a53be;

  --bs-btn-active-shadow:inset 0 3px 5px rgba(0, 0, 0, 0.125);

  --bs-btn-disabled-color:#fff;
  --bs-btn-disabled-bg:#0d6efd;
  --bs-btn-disabled-border-color:#0d6efd;
}

Important:
All values that start with --bs-btn are Bootstrap CSS variables. For example, --bs-btn-bg:#0d6efd stores the main blue color of the button.

This means:

  • The main button background is blue
  • The text color is white
  • On hover, the blue becomes darker
  • On click, the button looks pressed in
  • When disabled, the button keeps its color but becomes inactive

The idea behind this example:

.btn builds the button structure and spacing .btn-primary only supplies the colors and effects

So Bootstrap is not magic, it is just a very big ready-made CSS file.

Bootstrap Containers

1. What Is a Bootstrap Container

A Bootstrap container is a wrapper that holds and centers the content of the page. Almost every Bootstrap layout starts with a container.

The container controls:

  • The maximum width of the content
  • The side margins
  • The responsive behavior of the layout

2. Types of Bootstrap Containers


<div class="container">...</div>
<div class="container-fluid">...</div>
  • container, fixed max width, centered
  • container-fluid, always 100 percent width

3. container, Fixed Width Example

<div class="container">
  This is a fixed width container
</div>


4. container-fluid, Full Width Example

<div class="container-fluid">
  This is a full width container
</div>


5. Responsive Behavior

Resize your browser window and look at the difference:

  • The container stays centered and limited in width
  • The container-fluid always fills the entire screen
<div class="container">
   <!-- rows and columns go here -->
</div>


6. Summary

  • A container wraps the page content
  • container is centered and has max width
  • container-fluid is always full width
  • Every Bootstrap layout starts with a container

Bootstrap Grid View, How Layout Really Works

1. Basic Grid Structure

Every Bootstrap grid is built from three mandatory layers:

  1. container, wraps the entire layout
  2. row, creates a horizontal line
  3. col-*, creates the columns inside the row

This is the basic structure:

one of three columns
one of three columns
one of three columns

<div class="container">
  <div class="row">

    <div class="col-sm-4">one of three columns</div>
    <div class="col-sm-4">one of three columns</div>
    <div class="col-sm-4">one of three columns</div>

  </div>
</div>

Explanation:

  • container controls the outer width and centering
  • row activates the Flexbox grid system
  • col-sm-4 means: each column takes 4 parts out of 12

Since 4 + 4 + 4 = 12, the three columns fit perfectly on one row.

2. What Does col-sm-4 Mean

The class name col-sm-4 is made of three parts:

  • col = this is a column
  • sm = this layout starts from small screens and up
  • 4 = this column takes 4 out of 12 parts

So:

  • On small screens and larger, we get 3 columns in one row
  • On very small screens, they stack one under another automatically

3. Bootstrap Grid Breakpoints

Bootstrap changes layout based on screen width. These are the main breakpoints:

  • Extra small (xs), below 576px → col-
  • Small (sm), 576px and up → col-sm-
  • Medium (md), 768px and up → col-md-
  • Large (lg), 992px and up → col-lg-
  • Extra large (xl), 1200px and up → col-xl-

4. Max Container Width by Screen Size for Non-Fluid Container

Bootstrap container automatically changes its maximum width:

  • Small screens → about 540px
  • Medium screens → about 720px
  • Large screens → about 960px
  • Extra large screens → about 1140px

container → limited width, centered
container-fluid → unlimited width, full screen

Responsive Breakpoints, How Bootstrap Knows When to Change Layout

Bootstrap changes the layout automatically based on the screen width. It does this using CSS media queries, which check how wide the device screen is.

1. Extra Small Devices (Phones, Less Than 768px)


/* Extra small devices (phones, less than 768px) */
/* No media query since this is the default in Bootstrap */

This means:

  • Phones are the default layout in Bootstrap
  • No media query is needed for this size
  • On very small screens, columns usually stack vertically

2. Small Devices (Tablets, 768px and Up)


@media (min-width: @screen-sm-min) { ... }

This means:

  • When the screen becomes 768px wide or more
  • Bootstrap activates the small (sm) layout
  • Classes like col-sm-4 start working

3. Medium Devices (Desktops, 992px and Up)


@media (min-width: @screen-md-min) { ... }
  • Activated at 992px and wider
  • This is for laptops and regular desktops
  • Classes like col-md-6 become active

4. Large Devices (Large Desktops, 1200px and Up)


@media (min-width: @screen-lg-min) { ... }
  • Activated at 1200px and wider
  • This is for large computer screens
  • Classes like col-lg-3 become active

5. Using max-width to Target Only a Specific Range


@media (max-width: @screen-xs-max) { ... }

@media (min-width: @screen-sm-min) and (max-width: @screen-sm-max) { ... }

@media (min-width: @screen-md-min) and (max-width: @screen-md-max) { ... }

This allows Bootstrap to apply CSS:

  • Only on phones
  • Only on tablets
  • Only on laptops

Instead of saying “from this size and up”, we say “only in this exact size range”.

6. How This Affects Bootstrap Columns

When you write: 


<div class="col-sm-4 col-md-6 col-lg-3"></div>

Bootstrap behaves like this:

  • Below 768px, columns stack vertically
  • From 768px, it becomes col-sm-4
  • From 992px, it becomes col-md-6
  • From 1200px, it becomes col-lg-3

7. Summary

  • Bootstrap uses media queries to detect screen size
  • Each screen size has a name: xs, sm, md, lg
  • Each size activates different column layouts
  • This is what makes Bootstrap fully responsive

5. Summary

  • The Bootstrap grid is always based on 12 columns
  • container → row → col is the required structure
  • col-sm-4 means 3 equal columns starting from small screens
  • Breakpoints control when columns stack or stay horizontal
  • Everything is responsive automatically

Bootstrap Buttons

Bootstrap provides ready-made button styles that give visual meaning to actions. Each button color represents a different type of action such as primary, success, warning, or danger.

1. Live Button Examples

These are the exact button types shown in the slide:



2. Button Code Examples

This is how each button is created in HTML: 

<!-- Primary action -->
<button type="button" class="btn btn-primary">Primary</button>

<!-- Secondary outline button -->
<button type="button" class="btn btn-secondary">Secondary</button>

<!-- Success message or positive action -->
<button type="button" class="btn btn-success">Success</button>

<!-- Informational button -->
<button type="button" class="btn btn-info">Info</button>

<!-- Warning, attention required -->
<button type="button" class="btn btn-warning">Warning</button>

<!-- Dangerous or destructive action -->
<button type="button" class="btn btn-danger">Danger</button>

<!-- Looks like a link but behaves like a button -->
<button type="button" class="btn btn-link">Link</button>

3. What Does Each Button Mean

  • Primary, main action on the page
  • Secondary, alternative action
  • Success, successful or positive action
  • Info, informational message
  • Warning, caution required
  • Danger, delete, remove, or risky action
  • Link, looks like a link but acts as a button

4. Important Rule About Bootstrap Buttons

Every Bootstrap button is built from two classes together:

  • btn, activates basic button styling
  • btn-primary, btn-success, btn-danger, choose the color and meaning

Without btn, the button will NOT look like a Bootstrap button.

5. Summary

  • Bootstrap buttons come with built-in colors and meanings
  • They require two classes, btn and a color class
  • They save time and keep design consistent
  • The color of the button helps users understand the action

Adding Items to the Navbar and Changing Its Color

In Bootstrap, the navbar is built using a combination of HTML elements and predefined classes. We can easily add menu items, mark an item as active, disable items, and change the navbar color in several ways.

1. Basic Navbar With Items

This is a basic structure:

  • Active
  • Disabled 

<nav class="navbar navbar-dark bg-dark navbar-expand-lg">
  <ul class="navbar-nav">

    <li class="nav-item">
      <a class="nav-link active" href="#">Active</a>
    </li>

    <li class="nav-item">
      <a class="nav-link disabled" href="#">Disabled</a>
    </li>

  </ul>
</nav>

Explanation:

  • navbar, activates the Bootstrap navbar component
  • navbar-dark, makes the text suitable for dark backgrounds
  • bg-dark, gives the navbar a dark background color
  • navbar-expand-lg, the navbar expands horizontally on large screens
  • navbar-nav, wrapper for the menu items list
  • nav-item, one menu item
  • nav-link, the actual clickable link
  • active, marks the current page
  • disabled, disables the link visually and functionally

2. Changing the Navbar Color Using Bootstrap Classes

Instead of using bg-dark, we can change the background color using other Bootstrap background classes.

  • Active
  • Disabled 

<nav class="navbar navbar-light bg-light navbar-expand-lg">

This makes the navbar:

  • Light background
  • Dark text

Another example:

  • Active
  • Disabled 

<nav class="navbar navbar-light bg-primary navbar-expand-lg">
  • Primary blue background
  • Light text on the navbar

3. Changing the Navbar Color Using Custom CSS

We are not limited to Bootstrap colors only. We can also create our own custom background color using a CSS class.

  • Active
  • Disabled 

<nav class="navbar navbar-light navbar-expand-lg my-nav">

And define the color using CSS: 


<style>
  .my-nav {
    background-color: rgb(248, 25, 237);
  }
</style>

This method allows you to:

  • Use any custom color you want
  • Match your project branding
  • Override Bootstrap default colors safely

4. Important Rule About Navbar Colors

  • navbar-dark is used with dark backgrounds
  • navbar-light is used with light backgrounds
  • bg-* controls the background color
  • Custom colors can be applied using your own CSS class

5. Summary

  • Navbar items are added using ul → li → a
  • active highlights the current page
  • disabled blocks a link
  • Navbar colors can be changed using:
    • Bootstrap background classes
    • Custom CSS classes

Summary

  • Bootstrap is a ready-made design framework
  • We add it using a CDN link
  • Bootstrap works using predefined classes
  • No custom CSS is required for basic styling
