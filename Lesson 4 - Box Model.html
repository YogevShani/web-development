8. CSS Reset

Different browsers apply their own default styling (margins, padding, font sizes, line-height, list bullets and more). A CSS Reset removes these differences so your design starts from a clean, consistent baseline.

Why Use a CSS Reset?

  • Removes default browser margins and padding
  • Makes all browsers behave the same
  • Prevents unexpected spacing
  • Gives you full control over layout

Simple Reset (commonly used)


* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

This is the most common reset today. It removes all margins and padding, and makes sizing predictable using box-sizing: border-box.

Meyer Reset (classic full reset)


html, body, div, span, applet, object, iframe,
h1, h2, h3, h4, h5, h6, p, blockquote, pre,
a, abbr, acronym, address, big, cite, code,
del, dfn, em, img, ins, kbd, q, s, samp,
small, strike, strong, sub, sup, tt, var,
b, u, i, center,
dl, dt, dd, ol, ul, li,
fieldset, form, label, legend,
table, caption, tbody, tfoot, thead, tr, th, td {
  margin: 0;
  padding: 0;
  border: 0;
}

This reset removes nearly all default browser styles, creating a completely neutral starting point.

9Ô∏è‚É£ Margin Collapsing Visual

When two vertical margins meet, **only the larger one** applies. So if one element has margin-bottom: 40px and the next has margin-top: 20px, the total spacing between them will still be **40px**, not 60px.

Top Box (margin-bottom: 40px)
Bottom Box (margin-top: 20px)

Notice the space between them ‚Äî it's not 60 px but 40 px (the larger of the two margins).


.boxA { margin-bottom: 40px; }
.boxB { margin-top: 20px; }
/* Only the larger (40px) is applied between them */
