Understanding the HTML Viewport

1. What is the viewport

The viewport is the visible area of a web page inside the browser window. Different devices have different viewport sizes. A desktop screen is wide, a phone screen is narrow. If the page does not adjust to the device, the layout becomes too big or too small.

2. Why the viewport meta tag is needed

Without the viewport meta tag, mobile browsers zoom out automatically. The entire page becomes very small, text is hard to read, and the layout does not respond to screen size. The viewport meta tag tells the browser to match the page width to the device width and use normal zoom. 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

width=device-width tells the browser to set the page width equal to the device screen width. initial-scale=1.0 tells the browser to load the page with normal zoom, not zoomed out.

Why the image behaves correctly in the responsive page

The responsive example uses two important CSS rules: 


img {
    max-width: 100 percent;
    height: auto;
}

max-width: 100 percent tells the browser that the image should never be wider than its container. This allows the image to shrink when the screen is smaller than the image width. Without this rule the image keeps its original size and becomes wider than the phone screen.

height: auto preserves the correct shape of the image when it is resized. The browser adjusts the height automatically so the image does not stretch or get distorted.

Together these two rules make the image fully responsive and allow the text to flow below it in a readable layout.

5. Summary

  • Viewport defines the visible area of a page
  • Mobile browsers zoom out by default if no viewport is set
  • The meta viewport tag makes pages responsive
  • width=device-width fits the content to the device
  • initial-scale=1.0 loads the page at normal zoom
